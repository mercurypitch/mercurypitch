Merc — mascot explorations

Three directions for the hero mascot, sketched as hand-authored SVG on the brand palette (BRAND.md §5/§6.6). These are pre-visualizations to lock silhouette, face language and pose vocabulary before Higgsfield/Nano Banana look-dev renders the surfacing. All motion here is the exact technique that ships in-app: SVG + CSS keyframes, ~2 KB per character, themeable, reduced-motion-safe.

Hover a card — Merc celebrates ✦

A · Lumen droplet

BRAND.md §6.6 faithful — spectrum gradient body, chrome rim

The soft luminous droplet the brand doc already specifies. Serene by default, beams when you nail a note.

  • For: warmest read; same family as the nine voice specialists; simplest rig.

1 · Lock it with Higgsfield look-dev

  1. From an approval-capable Claude session (claude.ai chat / desktop / CLI — remote headless sessions can't approve connector calls), run generate_image with nano_banana_pro (and soul_2 as a B-side), count: 2 per direction, 1:1. Prompts: MASCOT.md Phase 0.
  2. Pick the winning direction, then iterate conversationally on the same character: ¾ view, side, back; idle / listening / celebrate / encouraging / singing; mouth shapes.
  3. Export the locked character sheet into docs/branding/mascot/ with a prompt log.

2 · Then make it alive

Rebuild the locked design as original vector (copyright-clean per BRAND.md §7) and rig it as a <Mascot state energy /> Solid component — the same SVG + CSS technique animating this page, wired to the live pitch signals: bob with playback, lean with your pitch, celebrate on Perfect, encourage when off. Landing hero uses the same component large; Higgsfield video loops are only for the pitch deck and socials.
